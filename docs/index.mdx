# Introduction

Blume is an open-source, Markdown-first documentation framework powered by Astro and Vite. It is designed to turn documentation content into a polished, production-ready site with very little application code or configuration.

## Why Blume?

A Blume project can start as a folder of Markdown files. Navigation is inferred from the content structure, while search, theming, SEO, and common documentation features are available by default.

This makes Blume a good fit for:

- Product and API documentation
- Open-source project documentation
- Internal knowledge bases
- Developer portals
- AI-readable documentation sites

## Quick start

Create a new project with the Blume CLI:

```bash
npx blume init
```

Install Blume in an existing JavaScript project:

```bash
npm install blume
```

A minimal configuration can look like this:

```ts title="blume.config.ts"
import { defineConfig } from "blume";

export default defineConfig({
  title: "My Documentation",
});
```

Once the project is initialized, add Markdown or MDX pages to the configured content directory and run the development server.

## Content sources

Blume reads local Markdown by default and can also combine content from other sources. The official site demonstrates filesystem, GitHub, Sanity, Notion, and custom source adapters.

```ts title="blume.config.ts"
import { defineConfig } from "blume";

export default defineConfig({
  content: {
    sources: [
      { type: "filesystem", root: "docs" },
      {
        type: "mdx-remote",
        github: {
          owner: "example",
          repo: "documentation",
          path: "docs",
        },
      },
    ],
  },
});
```

All sources are rendered through the same documentation components.

## Built for people and AI

Blume supports machine-readable documentation workflows out of the box:

- `llms.txt` can expose an index and corpus for language models.
- Pages can be retrieved as Markdown by appending `.md` to a URL.
- An MCP server can provide read-only search and page-reading tools to compatible agents.
- An in-page AI assistant can answer questions using a configurable model provider or OpenAI-compatible endpoint.

## Interactive API documentation

Drop an OpenAPI or AsyncAPI specification into a Blume project to render interactive API references with endpoints, schemas, and a “Try it” playground.

## Components

Blume includes accessible, theme-aware components for common documentation patterns, including:

- Callouts, cards, steps, tabs, and accordions
- Code groups and package-install tabs
- File trees, panels, tiles, and prompts
- Mermaid diagrams and KaTeX math
- Diffs, badges, type tables, and live component examples

Components are designed to be used directly from Markdown or MDX without requiring imports for every page.

## Production features

Blume includes features commonly needed when publishing documentation:

- Local search in development and production
- SEO metadata, Open Graph images, sitemap, RSS, and JSON-LD
- Dark mode and theming
- Internationalization with right-to-left support
- Changelog and blog support
- Custom pages and export options
